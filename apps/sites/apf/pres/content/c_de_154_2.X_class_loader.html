1. Einleitung

Ab Version 2.0 unterstützt das APF die native Nutzung von PHP-Namespaces. Analog zum Namespace-Konzept der Version 1.X werden alle Resourcen per Namespace addressiert (Klassen, Templates, Konfigurationen, ...). Jetzt eben etwas anders nach PHP-Namespace-Muster. Es ändert sich folgendes... Seit der Version 2.0 können Klassen, Templates und Konfigurationsdateien von unterschiedlichen Basis-Pfaden geladen werden. Dies ermöglicht Ihnen die Trennung von Framework- und Applikations-Code sowie die Trennung von produktivem Code und Test-Artefakten.

Hierzu stellt das APF einen ClassLoader-Mechanismus zur Verfügung, der auf Basis des PSR-0 Inhalte aus unterschiedlichen Bereichen laden kann. Die Konfiguration erfolgt in der zentralen Bootstrap-Datei und Sie können beliebig viele ClassLoader registrieren.
