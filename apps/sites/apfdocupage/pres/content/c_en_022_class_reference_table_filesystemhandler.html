Class reference table - filesystemHandler

The filesystemHandler implements helper methods for file and folder handling. The following list provides an overview of these functions:
  • changeWorkDir()
    Defines the current working directory.
  • renameFile()
    Renames a file.
  • deleteFiles()
    Deletes one or more files.
  • deleteFolderRecursive()
    Deletes a folder with it's subfolders and files included.
  • copyFile()
    Copies a file.
  • uploadFile()
    Handles a file uploaded via HTTP/PHP upload.
  • showDirContent()
    Shows the content (files, folders) of the current working directory.
  • showDirSize()
    Calculates the size of the current working directory.
  • showDirContentCount()
    Returns the amount of files in current working directory.
  • deleteWorkDir()
    Deletes the working directory.
  • showFileSize()
    Returns the file size of the file passed with the first argument.
  • showFileAttributes()
    Returns an array with the offsets 'Extension', 'FileName', 'Path' und 'FileBaseName'. If the file can be found on the filesystem also 'Size', 'ModifyingDate', 'DownloadTime' are included in the parameter array.
The parameters of each single method can be seen in the API documentation. To use this component it must be included by placing the following code in your application: import('core::filesystem','filesystemHandler'); The constructor of the class expects on optional parameter, that defines the current working dir. If not set during initialisation the woring dir can be changed using the changeWorkDir() method afterwards.
