Literature

It is recommended to read a reference guide on the following topics:
  • Page controller pattern
  • MVC pattern
  • Three tire architectur
  • Composite pattern
  • Frontcontroller pattern
  • Factory pattern
  • Service layer pattern
  • Abstract data mapper
  • Domain object model
With these knowledge, it is a lot easier to get a deep insight into the framework and the propagated software design. wikipedia can be characterized as a good ressource. In Addition, the web page of the university of Rostock provides a clearly arranged summary on design pattern.

Further information on designing and implementing software can be obtained in the whitepaper Entwurf und Implementierung eines Webportals zur Registrierung und Verwaltung von Weiterbildungsveranstaltungen auf Basis von MySQL und PHP (german only).

Beyond this online ressources the author advices the following books:
  • Patterns of enterprise application architecture - Martin Fowler (Reference guide)
  • Refactoring - Martin Fowler (Software refactoring und analysis)
  • UML distilled - Martin Fowler (UML guide with many real life examples)
  • Softwaretechnik Praxiswissen - Johannes Siedersleben (Reference guide on software development processes)
In most cases, the documentation, information or examples on this page is detailed enought to build your own website or web application. If there is any lack of documentation, don't hesitate to give feedback on that issue. For this you can use the contact form or the forum.
