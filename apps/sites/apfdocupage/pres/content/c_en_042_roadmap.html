Roadmap

The following sections describe the roadmap of the development and the documentation of the advanture php framework. It is based on the current planning of the core developer but can be influenced by feedback as well. To give feedback, please use the forum. The scheduling for the current release cycle is as follows:


 Release 2.0 (end of 2008)

Major features:
  • Disable support for PHP 4
  • Redesign to "PHP 5 only"
  • Integration of a workflow controller to easyily build wizzards

 Release 1.7 (~August 2008)

Major features:
  • Redesign of the documentation page
  • Introduction of the registry component to replace the ApplicationManager. This avoides the basic configuration and makes it much more easier to build bootstrap files.
  • Introduction of a generic or mapper component, that cannot only store objects in a relational database (like Table-Data-Gateway or Row-Data-Gateway) but also load and save complete object trees (Generic-Data-Mapper).
  • Based on the generic or mapper, a reusable user management modul should come.
  • Further tutorials (Behind the site) and translation of the rest of the documentation.
  • Delivery of seperated releases for PHP4 and PHP5.
  • Bugfixing & common improvements

 Release 1.6 (15.06.2008)

Major features:
  • Translation of the remaining tutorial chapters
  • Enhancement to the tutorials and beginner materials (front controller!)
  • Tutorial on the topics "front controller based login" and "image delivery with the front controller"
  • Add a SessionSingleton class to make objects available in singleton style during a session
  • Introduction of the connectionManager to add an abstraction layer on database access classes
  • Introduction if the SQLite database driver
  • Bugfixing & common improvements
  • Articel about coding conventions and decisions for rules
  • XML sitemap to improve search engine ranking (SEO)
  • Spam protection for the guestbook including update to the tutorial and the documentation
  • Documentation and delivery of the calendar component
  • CAPTCHA form control

 Release 1.5 (15.02.2008)

Release 1.5 merely is a documentation release. The original appointment (end of 2007) was not realistic at all, because the translation of the documentation was considered as important. Updates to documentation pages are done continuously, feature updates will be anounced on the start page. These versions will be availabe as beta preview versions as familiar with the last version.

Major features:
  • Translation of the documentation pages (done)
  • Translation of the tutorial pages
  • Installation of german and english forums (done)
  • Build a class reference table (done)
  • Change output and framework conponents to UTF-8 (is already successfully realized in an external project)
  • Modification to existing application to support multilanguage environment (done)
  • Create a tool to generate links from front controller action definitions (erledigt)
  • Spam protection for the comment module (done)
  • Bugfixing & common improvements

 Release 1.4 (25.10.2007)

To guarantee reliability of the release 1.4, it will be available as a beta version before official release date. This version will be compatible with PHP 5. New versions will be announced on the start page.

Major features:
  • Compatibility mode for PHP 5 plus operation on PHP 4 possible furthermore
  • Beginner tutorials based on "hello world!":
    • How do I create my first website with aid of the framework?
    • How can I integrate dynamic content to this site?
    • How can I achieve to load database content into the content area?
  • Enhancement of the mailSender component with the methodes setBCCRecipient() and addHeader().
  • Improvenments to the FAQs
  • Installation tutorial for use with the framework's code-only package.
  • Integrate social bookmarking tool (including tutorial).
  • Add-on to the form validator tags: within this release, the user should have the chance to add a generic validator to the form, that prints out a text enclosed by the tag definition, if the condition is not met. The tag must be available in valgroups and in normal areas of the form. Example:
    <form:genericval field="" button="" type="">Content, that should be printed</form:genericval>
  • Roadmap of the frameworks and documentation
  • Bugfixing & common improvements
