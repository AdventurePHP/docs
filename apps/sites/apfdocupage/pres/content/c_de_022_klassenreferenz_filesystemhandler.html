Klassen-Referenz - filesystemHandler

Der filesystemHandler implementiert Helfer-Funktionen für das Handling von Verzeichnissen und Dateien:
  • changeWorkDir()
    Setzt das aktuelle Arbeitsverzeichnis auf den im Argument angegebenen Wert.
  • renameFile()
    Benennt eine Datei um.
  • deleteFiles()
    Löscht Dateien im aktuellen Arbeitsverzeichnis.
  • deleteFolderRecursive()
    Löscht alle Inhalte eines Ordners.
  • copyFile()
    Kopiert eine Datei.
  • uploadFile()
    Handelt eine per HTTP hochgeladene Datei.
  • showDirContent()
    Zeigt den Inhalt des aktuellen Arbeitsverzeichnisses und gibt diese als Liste zurück.
  • showDirSize()
    Zeigt die Dateisystemgröße des aktuellen Arbeitsverzeichnisses.
  • showDirContentCount()
    Gibt die Anzahl der Dateien des aktuellen Arbeitsverzeichnisses aus.
  • deleteWorkDir()
    Löscht das aktuelle Arbeitsverzeichnis.
  • showFileSize()
    Gibt die Dateigröße der einer übergebenen Datei zurück.
  • showFileAttributes()
    Zeigt die Attribute 'Extension', 'FileName', 'Path' und 'FileBaseName'. Falls die Datei eine auf dem Filesystem befindliche Datei ist werden auch 'Size', 'ModifyingDate', 'DownloadTime', ... ausgegeben.
Die Parameter der einzelnen Methoden können in der API-Dokumentation eingesehen werden. Zur Verwendung der Klasse muss diese vor der Verwendung durch import('core::filesystem','filesystemHandler'); eingebunden werden. Bei der Initialisierung der Klasse erwartet der Konstruktor das gewünschte Arbeitsverzeichnis als Argument. Dieses kann jedoch auch später per changeWorkDir() geändert werden.
