Auf dieser Seite finden Sie einen Vergleich des APF zu anderen Frameworks. Es wird aufgezeigt, welche Features das APF herausragend gestaltet. Sie denken: "Das APF ist nur ein weiteres Framework"? Auf dieser Seite finden Sie Gründe, warum Sie das Adventure PHP Framework (APF) unbedingt einsetzen und sollten!

Das APF unterstützt Sie, enterprise-ready Web-Applikationen und -Module zu entwickeln, die schnell, sicher und wiederverwendbar sind. Das Framework bietet dabei sowohl erprobte Tools zur Entwicklung nach anerkannten Design-Pattern als auch fertige Module, die bekannte Aufgabenstellungen allgemeingültig lösen.

Die Features im Überblick

Die nachfolgende Liste zeigt Ihnen die Features nochmal im Detail und beschreibt diese kurz:

PHP 4

Das APF ist konzeptionell so abstrahiert, dass es weiterhin PHP in der Version 4 unterstützt. Gerade im Shared-Hosting-Bereich ist PHP 4 noch häufig im Einsatz.


PHP 5

Auf Grund der Konzeption wird PHP 5 ebenso unterstützt wir PHP 4. Für PHP 5 steht ein eigenes Release zur Verfügung in der die notwendigen Anpassungen an die geänderten Sprachkonstrukte vorgenommen wurden.


HMVC / MVC 2

MVC ist im Bereich Web ein sehr weit verbreitetes Design-Pattern. MVC für sich genommen bietet jedoch nur eine begrenze Anwendungsmöglichkeit. Aus diesem Grund findet im APF das erweiterte MVC-Konzept - HMVC bzw. MVC 2 genannt - Anwendung. Dieses ermöglicht nicht nur hirarchische Strukturen und noch flexiblere Implementierung, sondern garantiert auch einfache Erweiterbarkeit von bestehenden Anwendungen über TagLibs und Document-Controller!


Page-Controller

Der Page-Controller ist die zentrale Komponente des Frameworks. Diese stellt eine allgemeingültige Umgebung für die Entwicklung nach dem HMVC- bzw. MVC2-Pattern bereit. Der Page-Controller übernimmt dabei die Entgegennahme eines Requests, die Ausführung der Input-Filter, das Erzeugen des APF-DOM-Baumes, die Transformation desselben und die Ausgabe.
Durch den generischen Tag-Parser ist der Funktionsumfang des Page-Controllers sehr einfach erweiterbar und es gibt bereits heute zahlreiche Erweiterungen, die die Effektivität bei der Entwicklung deutlich steigern.


Front-Controller

Der Front-Controller gibt dem Entwickler die Möglichkeit, vor dem Erzeugen der Page-Controller-Seite Actions auszuführen. Diese können dazu genutzt werden um die Business-Schicht einer Applikation zu erzeugen und das Model dieser zu initialisieren. So ist es möglich, die Gestaltung der Präsentationsschicht durch die Businessschicht zu beeinflussen.
In Zusammenarbeit mit dem Page-Controller ist es mit dem APF möglich Module lose zu koppeln und damit für den Einsatz in unterschiedlichen Webseiten oder Applikationen zu implementieren.
Über das Front-Controller-Timing-Model ist es möglich Actions für die unterschiedlichsten Aufgaben einzusetzen.


TagLibs

Durch die generische Implementierung des Page-Controllers ist es möglich beliebige TagLibs zu definieren, wie sie aus dem JAVA-Bereich bekannt sind. Die Funktionalität ist dabei durch Klassen bestimmt, die einem definierten Interface genügen.
Der Entwickler kann dabei beliebige eigene TagLibs einbinden und so die Funktion des Frameworks erweitern oder die Anforderungen einer Anwendung umsetzen. Durch den Aufbau des APF-DOM-Baumes fügt sich jeder Tag als Kind-Objekt desjenigen Templates ein, in dem es definiert wurde. Da jedes Kind eine Beziehung zum Vater-Objekt besitzt ist es innerhalb einer TagLib möglich auf den kompletten DOM-Baum zuzugreifen und diesen zu manipulieren. Dies wird beispielsweise bei den Formular-TagLibs verwendet um Validierungsinformationen eines Formularelements an das Formular weiterzugeben.


Erweiterte Konfiguration

Das Framework kennt drei Unterscheidungsmöglichkeiten für die Adressierung von Konfigurationsdateien: Namespace, Context und Umgebung. Durch diese Parameter ist es nicht nur möglich Module für unterschiedliche Applikationen zu entwickeln, sondern auch für verschiedene Umgebungen (Stage, Live, ...).


Tag-basierte Formulare

Durch die Tag-basierte Definition von Formularen hat der Template-Bauer oder Entwickler die maximale Flexibilität hinsichtlich der Gestaltung von Formularen. Aufwändige Definitionen von Formularen in Controller-Dateien und unflexible Gestaltung von Formularen über PHP-Code entfallen gänzlich. Durch das <form:addtaglib />-Tag ist es einfach, die bestehende Formular-Funktion mit eigenen Taglibs zu erweitern.


Integriertes Benchmarking

Das Thema Benchmarking ist im Adventure PHP Framework durch die zentrale Komponente generisch abgebildet. Es werden auch ohne weitere Messpunkte die wichtigsten Punkte gemessen um bereits während der Entwicklung Performance-orientiert Software-Entwicklung betreiben zu können.


Erweitertes URL-Handling

Die Gestaltung der Input-Filter ermöglichen den Betrieb einer Applikation bzw. Webseite im normalen URL-Layout und mit Rewrite-URLs. Es stehen für beide Betriebsarten entsprechende Filter und Tools zur Generierung und Manipulation von URLs zur Verfügung.

Die Struktur und Komplexität des DOM-Baumes wird dabei durch den Entwickler definiert. Jeder Knoten entspricht dabei ein HMVC-Knoten, dessen Ausgabe über einen Controller oder über die Funktionalität von TagLibs beeinflusst werden kann. So muss die MVC-Struktur nicht bei jeder Applikation/jedem Modul neu implementiert werden.
