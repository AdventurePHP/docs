Die Quellen des Adventure PHP Framework werden in einem öffentlichen GitHub-Repository verwaltet. Von dort kann der aktuelle Entwicklungsstand sowie Fehlerbehebungen bezogen werden. Ebenso können Sie dort neue Funktionen bereits vor der Integration in den Haupt-Zweig testen.

Mit dem Release 2.1 wurde die Code-Verwaltung auf GIT umgestellt und das Repository zu GitHub umgezogen.

In den GitHub-Repositories finden Sie alle Bestandteile des Frameworks, der Dokumentation, Beispiel-Anwendungen (z.B. die ) und weitere Tools.

Bitte beachten Sie, dass Quellen aus dem master oder feature-*-Branches instabile Änderungen enthalten können. Nutzen Sie nach Möglichkeit daher die release-*-Branches. Sofern Sie neue Funktionalitäten ausprobieren möchten, sind Sie natürlich herzlich eingeladen, den aktuellen Entwicklungszweig zu nutzen.

Das code-Repository beinhaltet den Kern des Frameworks. Möchten Sie diesen lokal nutzen, genügt ein

mkdir APF && cd APF git clone https://www.github.com/AdventurePHP/code .

Anschließend können Sie wie gewohnt Ihre Anwendung betreiben.

Mehr zum Arbeiten mit GIT und dem APF erfahren Sie im Wiki unter GIT Repository und Lokales Setup APF-Doku-Seite. Eine sehr gute Zusammenfassung zum Arbeiten mit GIT findet sich auch unter symfony.com/doc/current/contributing/code/patches.html.
