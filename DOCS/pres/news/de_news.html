APF Seite geht offline!

Mittwoch, 27. Januar 2021

In den letzten beiden Jahren habe ich meinen Fokus auf neue berufliche und private Herausforderungen gelegt. Aus diesem Grund ist das letzte, offizielle Release auch bereits etwas "in die Jahre gekommen".

Ich habe insgesamt nur wenig Zeit gefunden um am APF weiter zu arbeiten und das wird sich auf absehbare Zeit auch nicht ändern. Mit einem lachenden (es war eine tolle Zeit!) und einem weinenden (es fällt mir nicht leicht!) Auge beende ich hiermit offiziell die Arbeit am APF.

Was bedeutet das?

  • Die offizielle APF-Seite wird Ende Februar offline gehen.
  • Die GitHub-Organisation und das Projekt werden inkl. Code weiter verfügbar sein.
  • Das Wiki ist bereits offline geschaltet und die Artikel wurden unter dem wiki-export Repository als .md Dateien zur Verfügung gestellt.
  • Das Forum ist bereits offline geschaltet und die Kommunikation kann bei Bedarf über das GitHub Issue Tracking weiter geführt werden.
  • Die Issues des Bug-Tracker wurden als GitHub-Issues importiert und sind hier verfügbar.
  • Die Dokumentation ist in Form von (Roh-)HTML-Dateien weiter unter AdventurePHP/docs verfügbar.

Ich bedanke mich bei allen, die das APF und mich über die letzten Jahre unterstützt und begleitet haben. Es war eine tolle Zeit und eine tolle Erfahrung für mich!

Bei Interesse an einer Weiterentwicklung freue ich mich auf eine Nachricht über GitHub, per E-Mail oder andere Kanäle.

Alles Gute & bleibt gesund!

Christian

APF 4.0 released!

Sonntag, 27. Januar 2019

Das APF Team ist stolz darauf heute die Version 4.0 des Adventure PHP Frameworks freizugeben.

Mit dem Release 4.0 setzt das APF nun mindestens die Version PHP 7.0.0 voraus. Im Rahmen der Umstellung auf PHP 7 wurden viele native Sprach-Konstrukte - u.a. Typisierung von Methoden-Parametern - eingeführt und die Code-Basis einer umfassenden Optimierung unterzogen.

Die wichtigsten Neuerungen im Überblick:

  • Mindestens PHP 7.0 notwendig!
  • Methoden-Signaturen sind typisiert!
  • mcrypt durch OpenSSL ersetzt.
  • GORM unterstützt beforeDelete()-Hook für Domänen-Objekte.
  • GORM unterstützt Tabellen-Präfixe.
  • <form:listener />-Tags lassen sich in <form:error /> verwenden.

Beim Umstieg auf die Version 4.0 sind auf Grund der komplett überarbeiteten API Anpassungen an Code-Dateien ihrer bestehenden Applikationen notwendig. Eine Anleitung und Hinweise finden Sie unter . Die komplette API finden Sie in der API-Dokumentation.

Die vollständige Roadmap lässt sich im Adventure PHP Framework Issue Management-Tool einsehen. Details zu den im Release erledigten Tickets finden Sie ebenfalls dort.

APFelSMS - das Adventure PHP Framework Site-Management-System

Mittwoch, 16. Januar 2013

Bei der Erstellung einer Webseite fallen immer wieder ähnliche Aufgaben an. Dazu gehört zum Beispiel die Erstellung einer Navigation oder das Füllen des title-Tags mit dem Titel der aktuellen Seite.

Mit dem APFelSMS - dem APF Site Management System - liegt dem APF in Version 1.16 erstmals eine Erweiterung bei, die dem Entwickler all diese Aufgaben abnimmt. Die Seitenstruktur wird dazu einmalig in einer XML-Baumstruktur abgelegt. Anhand dieser Daten erzeugt das APFelSMS Navigationen und Links, bindet zusätzliche Stylesheets oder Scripte ein und setzt den Seitentitel in das title-Tag ein.

Jede Seite lässt sich um zusätzliche Funktionen erweitern und so z.B. mit zusätzlichen Request-Parametern oder externen URLs ausstatten, verstecken oder mit Zugriffsrechten belegen.

Und sollte das APFelSMS einmal Ihren Anforderungen nicht gerecht werden, so können Sie die bestehende Funktionalität problemlos austauschen, erweitern oder neue problemlos integrieren.

Weitere Details erfahren Sie im APF Wiki.
