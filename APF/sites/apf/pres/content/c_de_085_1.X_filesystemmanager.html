Der FilesystemManager bietet Methoden zum Handling von Ordner und Dateien. Hierzu werden statische Methoden zum Löschen, Kopieren und Verschieben von Dateien und dem Anlegen und Löschen von Ordner. Wie der Roadmap zu entnehmen ist, beinhaltet die Version 1.8 das Redesign des FilesystemManager. Die neue Komponente wurde von Abhängigkeiten zum core-Namespace befreit und in den tools-Namespace verlagert.

Die statische Klasse bietet folgende Methoden:
  • deleteFolder(): Löscht einen Ordner. Falls der zweite Parameter true enthält, wird der Ordner rekursiv gelöscht.
  • createFolder(): Legt einen Ordner (rekursiv) an. Im zweiten Parameter kann die umask angegeben werden.
  • copyFile(): Kopiert eine Datei.
  • removeFile(): Löscht eine Datei.
  • uploadFile(): Behandelt eine per HTTP hochgeladene Datei.
  • renameFile(): Benennt eine Datei um.
  • getFolderContent(): Zeigt den Inhalt eines Ordners.
  • getFileAttributes(): Zeigt die Attribute einer Datei.
  • getFolderSize(): Gibt die Größe eines Ordners in Byte zurück.

Details zu den Parametern und Rückgabewerten kann der API-Dokumentation des jeweiligen Releases unter eingesehen werden. Vor der Verwendung der Klasse muss diese per

import('tools::filesystem','FilesystemManager');

eingebunden werden.
